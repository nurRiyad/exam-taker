# ADR 0003: Treat Question Entry As A Core Product Surface

## Status

Accepted

## Context

The strongest adoption risk is not whether MCQ exams can be rendered. The risk is whether teachers can create exams quickly enough to keep using the platform.

## Decision

Question entry is a first-class product feature, not an admin afterthought.

## Consequences

MVP must support:

- Fast keyboard-friendly question creation.
- Mobile-compatible question creation.
- Bangla and English text.
- Topic/subject tagging.
- Correct option selection.
- Optional explanation.
- Draft saving.

Likely next features:

- Bulk paste from structured text.
- CSV/XLSX import.
- Question duplication.
- Reuse from previous exams.
- Image attachments if teacher interviews prove it is common.
